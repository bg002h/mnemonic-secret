# ms `mnem` wordlist-language hint — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (or executing-plans). Steps use checkbox (`- [ ]`) syntax.

**Goal:** Put the BIP-39 wordlist language on the `ms1` wire via a new byte-aligned `mnem` payload kind (`[0x02][lang_byte][entropy]`), fixing the §6.3 non-English-seed footgun; `ms encode` auto-emits `mnem` for non-English, English/`--hex` stay byte-identical `entr`.

**Architecture:** `mnem` is a new prefix-byte-discriminated payload (`0x02`) that goes through the SAME byte-oriented `from_seed`/`Parts::data()` path as v0.1 entr (no bit-path — byte-aligned). The decoder's prefix-byte dispatch becomes load-bearing. Three v0.1-hardcoded gates (a length check + two CLI handlers) must learn the `0x02` arm.

## Phase 0 — spike: prove all 5 lengths construct (R0 C1 gate; throwaway)

> The bit-aligned 4-bit design was unconstructible for 3/5 lengths (codex32 `sanity_check`). This spike proves the byte-aligned layout works for ALL five BEFORE any real work, and pins the length set. A 12-or-21-word-only fixture would re-hide C1 — test all five.

**Files:** Test: `crates/ms-codec/tests/spike_mnem_lengths.rs` (throwaway; delete or keep as a guard at Phase 1).

- [ ] **Step 1: Write the spike.**
```rust
use codex32::{Codex32String, Fe};
#[test]
fn mnem_byte_aligned_constructs_for_all_five_lengths() {
    let mut lens = vec![];
    for n in [16usize, 20, 24, 28, 32] {
        let mut data = vec![0x02u8, 0x00u8]; // prefix + lang(English=0)
        data.extend(std::iter::repeat(0xABu8).take(n));
        let c = Codex32String::from_seed("ms", 0, "entr", Fe::S, &data)
            .unwrap_or_else(|e| panic!("N={n} failed sanity_check: {e:?}")); // must NOT panic for any n
        let s = c.to_string();
        // round-trips via the byte path?
        let back = Codex32String::from_string(s.clone()).unwrap().parts().data();
        assert_eq!(back[0], 0x02, "N={n} prefix");
        assert_eq!(back[1], 0x00, "N={n} lang");
        assert_eq!(&back[2..2 + n], &data[2..], "N={n} entropy");
        lens.push(s.len());
    }
    assert_eq!(lens, vec![51, 58, 64, 70, 77], "VALID_MNEM_STR_LENGTHS"); // pin (correct if spec right)
}
```

## Phase 3 — lockstep: toolkit FAITHFUL derive-and-preserve (user-chosen 2026-06-01)

> **Design change vs the original Phase 3 (which was SPEC-literal "toolkit emits entr only").** The user chose **faithful derive + preserve** + **develop-now-ship-together**. The toolkit must (a) **derive** every key from a mnem card using the **wire** language (not `--language`), and (b) **preserve** the language by **emitting `mnem`** whenever the run's effective language is non-English — so a non-English source round-trips through `bundle`/`convert` to a self-describing card instead of a language-losing `entr` card. The English path stays **byte-identical**. This EXPANDS the toolkit emit surface (the SPEC said emit-entr-only); the expansion is the whole point of "preserve" and is gated by THIS section's R0. **Phase-3 design R0 gate: GREEN** — R0 (2C/5I) → R1 (1C/2I) → R2 (0C/2I) → **R3 (0C/0I)**; reviews `design/agent-reports/ms-mnem-phase-3-design-R{0,1,2,3}-review.md`. Cleared for toolkit code.

**Core concept — language is a PER-CARD property, not a per-run global (R0-C1).** Each `ms1` card carries (mnem) or implies (entr) its own language; a multisig bundle can have cosigner cards in DIFFERENT languages, and the per-cosigner derive loops (`verify_bundle.rs:2411`, `wallet_import/overlay.rs:161`) take a single `language` today. So the unit of resolution is the card, not the run. Define a helper in `cmd` or `language.rs`:
```rust
// the language to use for THIS card's seed derivation / phrase reconstruction
fn payload_bip39_language(payload: &ms_codec::Payload, cli: CliLanguage) -> Result<bip39::Language> {
    match payload {
        ms_codec::Payload::Mnem { language, .. } => language::wire_code_to_bip39(*language), // wire wins
        _ /* Entr */ => Ok(cli.into()),                                                       // --language / English default
    }
}
```
Every site that **derives from a decoded card** calls this with THAT card's payload — inside per-cosigner loops, per cosigner. Single-source commands (xpub-search from one ms1, silent-payment, convert-from-one-phrase) have exactly one card so a single value is fine THERE; the multi-cosigner sites must resolve per card. **Wire-wins precedence:** when a `mnem` card's wire language is overridden by a conflicting explicit `--language`, emit a stderr `note:` (deduplicated across cosigners — one note per distinct overridden language), mirroring ms-cli decode's R0-M1.

**The carrier (R1-C1-RESIDUAL / R2-I6 — the sharp structural fix, corrected dataflow).** The `payload_bip39_language(payload, cli)` helper suffices where decode + derive are CO-LOCATED (xpub-search, silent-payment, the verify_bundle/overlay per-cosigner loops, convert's in-arm derive). It does **not** cover the `bundle --import-json` path, where the ms1 decode (`bundle.rs:1613`) and the ms1 re-emit (`synthesize_descriptor`, `synthesize.rs:291`) are in different functions across the `ResolvedSlot` (= `CosignerKeyInfo`, `synthesize.rs:616-637`) boundary, and the slot is built at `bundle.rs:1626` discarding the wire language. **Fix: add `pub language: Option<bip39::Language>` to `ResolvedSlot`** with semantics **"`Some(wire)` = this slot's source was a `mnem` card whose wire language OVERRIDES `--language`; `None` = defer to the run `--language`/English."** Resolution everywhere is `slot.language.unwrap_or_else(|| args.language().into())`.
> **R2-I6 dataflow precision (do NOT misread):** the `--import-json` flow does **NOT re-derive** — each cosigner's xpub comes from the envelope's mk1 chunk (`json_envelope.rs:364`), not from re-deriving the ms1 entropy — so for import-json the language is **emit-only**: populate `slot.language` at `bundle.rs:1626` from the decoded payload and READ it ONLY at the emit `synthesize.rs:291`. The SEPARATE `resolve_slots` flow (`bundle.rs:448-709`, source = phrase/entropy/seedqr/`--slot @N.entropy=<hex>`) has **no `ms_codec::decode`** and emits via `synthesize_unified` (`synthesize.rs:778`); its slots are `language: None` (no wire source) → derive (`bundle.rs:626/1369`) + emit resolve through `--language`. A non-English **phrase** bundle therefore still emits `mnem` (via the `None`→`--language` resolution being non-English), closing the phrase-source footgun, while `bundle.rs:626` does NOT read a non-`None` `slot.language`.
**(R2-M8) Constructor cascade:** adding the field compile-breaks **~28 `ResolvedSlot {…}`/`CosignerKeyInfo {…}` literal sites** (foreign-wallet parsers `coldcard.rs:336`, `bsms.rs:255`, `electrum.rs:397`, `sparrow.rs:433`, `specter.rs:251`, `bitcoin_core.rs:294`, `coldcard_multisig.rs:485`, `json_envelope.rs:364`, the two `pipeline.rs`, `synthesize.rs:936/958/1289/1409/1521`, `bundle.rs:525/595/642/685/1398/1453`, `verify_bundle.rs:856`, `parse_descriptor.rs:1249/1818`, + tests). **All of them set `language: None`** EXCEPT the import-json `mnem`-decode arm at `bundle.rs:1626` — every foreign-wallet / descriptor-concrete / hex slot carries no ms1 wire language. The compiler forces each; expect the cascade, don't mistake the 1-2 Step-4 sites for the whole set.

**Emit rule — also PER-CARD (the new behavior).** When the toolkit emits an `ms1` card from a given source's entropy+language: `lang == English` → `ms_codec::Payload::Entr(entropy)` (**BYTE-IDENTICAL to today** — the back-compat invariant, the overwhelmingly common case); `lang != English` → `ms_codec::Payload::Mnem { language: cli_language_to_wire_code(lang), entropy }` (NB the field is `language`, not `code` — R0 citation fix). So: mnem-ja source → mnem-ja card (preserve); non-English **phrase** `--language ja` → mnem-ja card (also closes the footgun for phrase sources — confirmed in-scope, R0-M2); English source → entr (no regression). In a multi-cosigner bundle each emitted ms1 card's kind is decided from THAT slot's source language (ja cosigner → mnem-ja card; en cosigner → entr).
