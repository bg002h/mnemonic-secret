## Critical

None.

The four Critical-class hazards the brief enumerated were each disproven by an executed probe:

- **Leaking share scheme** — disproven. The (k−1) defining shares are CSPRNG-filled (`shares.rs:139-141`, `getrandom::fill` into a `Zeroizing` buffer of the secret's byte length), independent of the secret. A below-threshold subset cannot recover (probe `p6_below_threshold_cannot_recover`: 2-of-(3-of-5) → `Err`; raw `interpolate_at` of 2 shares of a threshold-3 set → `ThresholdNotPassed`). The secret-at-S (index `s`) is never in the returned set (`shares.rs:147-155` distributes only `defining[1..]` + interpolated non-`s` indices; probe `p1b_secret_s_not_distributed` confirms the recovered secret-at-S string is byte-distinct from every distributed share and carries index `s`).
- **Non-recoverable set** — disproven. Exhaustive probe `p1_exhaustive_recover_all_ksubsets`: every (kind∈{entr,mnem}) × (n_ent∈{16,20,24,28,32}) × (k∈2..=9) × (n∈{k,k+1,k+3}≤31) × **every** C(n,k) subset → `combine_shares` returns the exact original `Payload`. Returns exactly `n` shares with distinct non-`s` indices and a single shared id.
- **Byte-identity break** — disproven. Probe `p4_byte_identity_all_lengths`: `encode_shares(tag, ZERO, 1, &p) == vec![encode(tag,&p)]` for entr+mnem × all 5 lengths; the ZERO path keeps `id = tag` (not random), threshold `'0'`, index `'s'`. The `package`→`payload_wire_bytes` extraction is a pure refactor (the byte construction at `envelope.rs:200-216` is the same `[prefix]||payload` logic moved verbatim); `vectors` + `round_trip` + `bip93_*` suites pass unchanged (full `cargo test -p ms-codec` green, 7 test binaries + lib + doc).
- **Tag-from-id bug** — disproven. `combine_shares` (`shares.rs:236-242`) recovers the secret-at-S via `interpolate_at(&parsed, Fe::S)`, then routes its bytes through `dispatch_payload` (header-gate-free, prefix→Payload only) and **hard-codes `Tag::ENTR`**, discarding the share-set's random id. It does NOT call `discriminate`. Probe `p3b_mnem_language_survives_tag_entr`: a mnem split (langs 0,1,2,5,9) recombines to `Payload::Mnem{language,…}` with the language preserved and `tag == Tag::ENTR`.

## Confirmations (incl. probe results)

**Construction correctness (`encode_shares`, shares.rs:101-159):**
- secret-at-S built via `from_seed(HRP, k, id, Fe::S, &bytes)` (:130); k−1 defining shares at the FIRST k−1 non-`s` pool indices with CSPRNG fillers (:138-143); interpolation fills the remaining `n−(k−1)` indices via `interpolate_at(&defining, pool_idx)` (:152-154). Defining indices `[0,k-1)` and interpolation indices `[k-1,n)` are disjoint slices of the 31-entry `non_s_index_pool()` → all `n` distributed indices distinct, none `s`. `n ≤ 31` (`:122`) guarantees the pool never exhausts. Returns exactly `n` (probe + `debug_assert_eq!` :157).
- `random_id()` (:40-55): 4 codex32-alphabet chars from `getrandom::fill`-derived bytes masked `& 0x1f`, re-rolling while `RESERVED_ID_BLOCKLIST.contains(&id)` (loop terminates w.p. 1; blocklist is 5/65536 of the id space). `String::from_utf8` cannot fail (alphabet is ASCII). Probe `encode_shares_id_not_in_blocklist` (64 iters) + `p6b_splits_are_randomized` (two splits of the same secret differ) confirm randomness + blocklist avoidance.
- Bounds: `Threshold::new` admits 2..=9 only (`:72-78`, codex32 rejects `1`); `ZERO` is a const, not `new(0)`. `encode_shares` ZERO path requires `n==1` (`:112-114`); share path requires `k ≤ n ≤ 31` (`:122`). Probes hit `InvalidShareCount{k:0,n:2}`, `{k:2,n:1}`, `{k:2,n:32}`.
- **getrandom error handling:** `getrandom::fill(...).expect("getrandom::fill must not fail")` (:43, :140). This panic-on-OS-CSPRNG-failure matches the cited `mk_codec` precedent (`mnemonic-key/.../string_layer/pipeline.rs:47`: `getrandom::getrandom(&mut buf).expect("OS CSPRNG must be available for mk1 encode")`). OS entropy failure is a process-fatal condition; panic is the codec convention. `getrandom` resolves to 0.3.4 in `Cargo.lock` (the `getrandom::fill` 0.3.x API; build + run succeed). Acceptable.

**`combine_shares` validation order + C1 (shares.rs:180-243):**
- Order matches the doc + plan Task 1.4: (1) parse each share (`from_string` → `Error::Codex32`); (1a) empty → `ThresholdNotPassed{1,0}`; (2) **reject any index-`s` input → `SecretShareSuppliedToCombine` BEFORE `interpolate_at`** (:207-209); (3) `parsed.len() < k` → `ThresholdNotPassed` (:215-221), `k` read from `fields[0].0 - b'0'`; (4) exhaustive distinct-index pre-check → `RepeatedIndex` (:225-232, codex32's own check is lazy per `interpolate_at` :283); (5) `interpolate_at(&parsed, Fe::S)` (:236) surfaces `Mismatched{Hrp,Id,Threshold,Length}`.
- **C1 is correctly motivated and placed.** codex32 `interpolate_at` short-circuits at `lib.rs:259-262` (`if indices[i] == target { return Ok(shares[i].clone()) }`) — returns a header-consistent index-`s` input directly, bypassing the Lagrange path AND the per-input payload validation. The spike test `interpolate_short_circuits_on_index_s` proves this empirically (incl. `[secret_s, secret_s]` → Ok, no `RepeatedIndex`). `combine_shares` pre-rejects index-`s` (:207) so this short-circuit can never smuggle an attacker-chosen secret-at-S past validation.
- **C1 ordering probe** (`p2_c1_index_s_order`): a SINGLE secret-at-S input (count 1 < k=2) → `SecretShareSuppliedToCombine`, NOT `ThresholdNotPassed` — confirming the index-`s` check (step 2) precedes the count check (step 3), as specified.
- **Note (safe, expected):** a v0.1 single-string passed to `combine_shares` → `SecretShareSuppliedToCombine` (probe `p3_v01_single_into_combine`), because a `'0'`-threshold string necessarily has index `s` (codex32 `parts_inner:237` rejects threshold-0 + non-`s`), so the C1 check catches it first. The doc/plan narrate this case as "ThresholdNotPassed", but the actual `SecretShareSuppliedToCombine` is a strictly safe and more precise rejection (the v0.1 single's index genuinely IS `s`). Behavior is correct; only the doc-comment's predicted error name (`shares.rs:213-214`) is slightly off. Cosmetic — not even raised to Minor.
- The threshold-byte `fields[0].0 - b'0'` arithmetic is safe: every parsed share has a threshold byte ∈ `{'0','2'..'9'}` (codex32 `parts_inner` validates on parse), so the subtraction yields `{0,2..9}` and `k=0` (only for a `'0'` single) is unreachable here because step 2 already rejected it. Wire offsets in `extract_wire_fields` are `sep + N` (separator-relative), so they are correct for both short and long brackets (all ms1 strings are short bracket: max entr-32 share = 75 < 81).

**`dispatch_payload` + discriminate refactor + decode routing (envelope.rs:95-189):**
- `dispatch_payload` (:196-187 region) is header-gate-free: reads `data[0]` → `0x00`→`Entr(rest)` / `0x02`→`Mnem{language:rest[0],entropy:rest[2..]}` + `validate()` / else `ReservedPrefixViolation`. `discriminate`'s tail now calls it (b'0' path unchanged; `discriminate_decodes_*` tests pass). The threshold dispatch (:110-127) is `THRESHOLD_V01('0')`→proceed (share-index check stays ONLY here) / `b'2'..=b'9'`→`IsShareNotSingleString{threshold,index}` / `other`→`ThresholdNotZero{got}`. Matches plan Task 1.5 verbatim.
